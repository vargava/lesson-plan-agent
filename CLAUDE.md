# Lesson Plan Agent

## What This Project Does
A weekly agent that reads a Google Sheet containing a math curriculum unit, extracts two lesson tabs per run, generates detailed publication-ready lesson plans, and writes the output to a new Google Doc.

Claude Code is the model тАФ no separate Anthropic API key is needed. The two helper scripts handle Google API I/O; Claude Code does the lesson generation directly.

## Architecture
```
Claude Code
  тФЬтФАтФА python src/fetch_lessons.py  тЖТ reads Google Sheets, outputs lesson data as JSON
  тФЬтФАтФА [Claude Code generates Lesson A using SKILL.md + lesson data]
  тФЬтФАтФА [Claude Code generates Lesson B using SKILL.md + lesson data]
  тФФтФАтФА python src/write_doc.py      тЖТ creates formatted Google Doc via Docs + Drive APIs
        тФФтФАтФА uses src/format_doc.py тЖТ parses lesson text into Docs API batchUpdate requests
```

## File Structure
- `src/fetch_lessons.py` тАФ reads next two lesson tabs from Google Sheets, prints JSON to stdout
- `src/write_doc.py` тАФ creates a richly formatted Google Doc from two lesson plan text files
- `src/format_doc.py` тАФ parses lesson plan plain text into Google Docs API formatting requests
- `src/auth_setup.py` тАФ one-time OAuth2 setup script; run once to generate `credentials/token.json`
- `SKILL.md` тАФ the full instructional design spec that drives lesson plan generation
- `state.json` тАФ tracks which lesson was last completed (auto-generated, never committed)
- `credentials/` тАФ Google credentials live here (never committed):
  - `service_account.json` тАФ used by fetch_lessons.py to read Google Sheets
  - `oauth_credentials.json` тАФ OAuth2 Desktop App credentials for write_doc.py
  - `token.json` тАФ auto-generated by auth_setup.py; never committed
- `.env` тАФ Google config (never committed)
- `tmp/` тАФ temporary files for lesson plan text during a run (auto-created, never committed)

## Google Auth тАФ Two Separate Credential Types
This project uses two different Google credential types for different reasons:

**Service account** (`credentials/service_account.json`):
- Used only by `fetch_lessons.py` to read Google Sheets
- The Sheet must be shared with the service account email
- Service accounts cannot own files in a personal Google Drive (no storage quota)

**OAuth2 user credentials** (`credentials/token.json`):
- Used by `write_doc.py` to create Google Docs
- Files are created as the authenticated user, using their Drive quota
- One-time setup: run `python src/auth_setup.py` to open a browser auth flow
- Requires `credentials/oauth_credentials.json` (Desktop App type from Google Cloud Console)
- Token auto-refreshes; only needs re-auth if revoked

**Note:** Domain-wide delegation is not applicable тАФ this is a personal Google account,
not a Google Workspace account.

## Google Doc Output Format
- A new Google Doc in the configured Drive folder
- Named: `YYYY-MM-DD тАФ [Lesson A tab name] & [Lesson B tab name]`
- Two lessons separated by a page break
- Rich formatting applied via Docs API batchUpdate:
  - Section headers тЖТ Heading 2
  - Time block headers (BLOCK X тАФ ...) тЖТ Heading 3, dark blue
  - [EN] sections тЖТ blue text
  - [ES] sections тЖТ green text
  - [MX] sections тЖТ orange text
  - Teacher script paragraphs тЖТ italic
  - Role labels (Teacher:, Students:, Discussion prompts:) тЖТ bold
  - Quality check items ([тЬУ]) тЖТ dark green bold

**Note on Google Docs tabs:** The Docs API does not currently support programmatic tab
creation via batchUpdate. Both lessons are written to a single tab with a page break.
Revisit when the API adds tab support.

## How Lesson Sequencing Works
- The Google Sheet has one tab per lesson
- Tab names encode order (e.g. "Week 12 - Lesson 5 - Ratios")
- Tabs whose names contain "overview", "unit", "index", "contents", "summary", or "cover" are skipped
- `state.json` stores `{"last_completed_lesson": "<tab name>"}` тАФ missing file means start from tab 1
- After a successful run, state.json is updated to the second lesson processed

## Workflow: Run the Lesson Plan Pipeline

When asked to run the lesson plan workflow, follow these steps exactly:

### Step 1 тАФ Fetch lesson data
```bash
python src/fetch_lessons.py
```
Parse the JSON output. It contains:
- `tab_a`, `tab_b` тАФ lesson tab names
- `data_a`, `data_b` тАФ raw lesson content as pipe-delimited cell grids (or sample text in dry-run)
- `dry_run` тАФ boolean

### Step 2 тАФ Read SKILL.md
Read `SKILL.md` from disk. This is the instructional design framework тАФ follow it exactly when generating lesson plans.

### Step 3 тАФ Confirm your understanding of the sheet structure
Before generating any lesson plans, briefly state your interpretation of the two lesson tabs тАФ their place in the unit sequence, the standards covered, the lesson objective, and any vocabulary or materials called out in the Unit Plan tab. Flag any ambiguities you see in the raw cell data. This gives the operator a chance to correct your understanding before you write 8000 words.

### Step 4 тАФ Generate Lesson A
Using the SKILL.md framework and `data_a` from Step 1, generate a complete publication-ready lesson plan for Lesson A.

**Note on input format:** The lesson data is a raw export from a Google Sheets lesson tab rendered as a pipe-delimited cell grid. Merged cells appear as empty strings. Cross-reference the Unit Plan tab data (available in Step 1's context from previous runs, or fetch if needed) to fill in lesson objective, vocabulary, materials, and exit ticket reference. Interpret the structure contextually.

Save the output to `tmp/lesson_a.txt` (create the `tmp/` directory if it doesn't exist).

### Step 5 тАФ Generate Lesson B
Same as Step 4 but using `data_b`. Save to `tmp/lesson_b.txt`.

### Step 6 тАФ Write the Google Doc

Before running the script, construct a human-readable `--title` from the lesson content:
- **Unit number** тАФ extract from the tab name prefix (e.g. `L2` тЖТ `Unit 2`), confirm against lesson content
- **Lesson numbers** тАФ look for lesson number labels in the sheet data (e.g. "Lesson 5"); cross-reference the Unit Plan tab if ambiguous
- **Topic name** тАФ from the unit topic in the lesson content or Unit Plan tab (e.g. "Fractions")
- Format: `Unit {N}: Lessons {X} and {Y} ({topic})`

```bash
python src/write_doc.py \
  --tab-a "<tab_a from Step 1>" \
  --tab-b "<tab_b from Step 1>" \
  --plan-a tmp/lesson_a.txt \
  --plan-b tmp/lesson_b.txt \
  --title "Unit 2: Lessons 5 and 6 (Fractions)"
```
If `dry_run` was true in Step 1, add `--dry-run` flag.

If a doc with the same title already exists in the Drive folder, `write_doc.py` will automatically append a suffix (e.g. `(2)`) and log a warning to stderr.

### Step 7 тАФ Report
Print a summary:
- Lesson A tab name
- Lesson B tab name
- Doc URL (from write_doc.py output)

## Key Constraints
- SKILL.md is the source of truth тАФ read it fresh from disk every run, never rely on memory of its contents
- Credentials folder is never committed
- state.json lives at project root, created on first run if absent
- fetch_lessons.py uses a service account; write_doc.py uses OAuth2 user credentials
- Generate lessons sequentially (A then B), not in parallel
